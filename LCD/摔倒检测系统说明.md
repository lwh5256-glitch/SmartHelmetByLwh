## 🎯 摔倒检测算法原理

### 检测流程

```
正常状态 → 自由落体 → 冲击 → 静止 → 检测到摔倒
  ↑                                      ↓
  └──────────────── 冷却期 ──────────────┘
```

### 状态说明

1. **正常状态 (Normal)**
   - 加速度模长正常 (约16384, 即1g)
   - 等待自由落体信号

2. **自由落体 (FreeFall)**
   - 加速度突然减小 < 8000 (约0.5g)
   - 表示开始下落

3. **冲击 (Impact)**
   - 加速度突然增大 > 25000 (约1.5g)
   - 表示撞击地面

4. **静止 (Stable)**
   - 加速度变化很小
   - 持续1秒以上

5. **检测到摔倒 (FALL!)**
   - 触发蜂鸣器报警
   - 进入3秒冷却期

### 滤波算法

使用**一阶低通滤波器**:
- 滤波系数 α = 0.3
- 公式: `y[n] = α * x[n] + (1-α) * y[n-1]`
- 作用: 去除高频噪声,平滑数据

## 🔔 蜂鸣器报警

- **检测到摔倒时**: 蜂鸣器以500ms间隔鸣叫
- **报警时长**: 3秒
- **冷却时间**: 3秒 (避免重复报警)

## 🧪 测试方法

### 1. 静止测试
- 水平放置传感器
- 观察 `Accel Mag` 应该在 16000-17000 之间
- `State` 应该显示 `Normal`

### 2. 模拟摔倒测试
- 将传感器从约30cm高度自由落下到软垫上
- 观察状态变化: Normal → FreeFall → Impact → Stable → FALL!
- 蜂鸣器应该开始报警

### 3. 调整灵敏度

如果误报或漏报,可以修改 `fall_detect.h` 中的阈值:

```c
#define FALL_ACCEL_THRESHOLD_HIGH   25000   /* 增大=降低灵敏度 */
#define FALL_ACCEL_THRESHOLD_LOW    8000    /* 减小=提高灵敏度 */
#define FALL_STABLE_TIME_MS         1000    /* 增大=更严格 */
```

## ⚠️ 注意事项

1. **蜂鸣器引脚**: PF8已配置为输出,不要用作输入
2. **采样频率**: 50ms (20Hz),足够检测摔倒
3. **电源稳定**: 确保MPU6050供电稳定,否则数据会抖动
4. **安装位置**: 建议将传感器固定在身体躯干位置
