本文只说明摔倒检测子系统，包括 MPU6050 数据来源、滤波、状态机、蜂鸣器联动和显示内容。整机接线以 `硬件连接总表与扩展规划.md` 为准。

## 4. 硬件连接摘要

| 模块 | 引脚 | 说明 |
|---|---|---|
| MPU6050 SCL | `PB6` | 软件 I2C |

| 蜂鸣器 | `PF8` | 推挽输出 |
| LCD | SPI LCD | 用于显示状态和数据 |

## 5. 状态机流程

```text
Normal -> FreeFall -> Impact -> Stable -> Detected
   ^                                      |
   |----------- cooldown/reset -----------|
```

| 状态 | 含义 |
|---|---|
| `FALL_STATE_NORMAL` | 正常监测，等待自由落体特征 |
| `FALL_STATE_FREEFALL` | 加速度模长低于下限阈值 |
| `FALL_STATE_IMPACT` | 加速度模长高于冲击阈值 |
| `FALL_STATE_STABLE` | 冲击后进入相对稳定判断 |
| `FALL_STATE_DETECTED` | 判定摔倒，触发报警 |

## 6. 当前阈值

以 `Drivers/HARDWARE/FALL_DETECT/fall_detect.h` 当前定义为准：

| 参数 | 当前值 | 说明 |
|---|---:|---|
| `FALL_ACCEL_THRESHOLD_HIGH` | `17000` | 冲击阈值 |
| `FALL_ACCEL_THRESHOLD_LOW` | `10500` | 自由落体阈值 |
| `FALL_STABLE_THRESHOLD` | `4000` | 静止判断阈值 |
| `FALL_STABLE_TIME_MS` | `350` | 静止持续时间 |
| `FALL_DETECT_COOLDOWN_MS` | `3000` | 检测冷却时间 |

说明：旧文档中的 `25000/8000/1000ms` 阈值已不对应当前代码，保留在历史记录中，不再作为调试依据。

## 7. 主循环调度

当前 `main.c` 中每约 `50ms` 执行一次核心读取与更新：

1. 读取 MPU6050 三轴加速度。
2. 调用 `fall_detect_update()` 更新状态机。
3. 刷新 LCD 上的 `Accel X/Y/Z` 和状态机状态。
4. 若进入 `FALL_STATE_DETECTED`，显示 `FALL!!` 并控制蜂鸣器鸣叫。

## 8. 告警优先级

当前蜂鸣器控制逻辑：

| 条件 | 行为 |
|---|---|
| 摔倒检测状态为 `FALL_STATE_DETECTED` | 最高优先级，蜂鸣器间歇鸣叫 |
| 未摔倒但 `PM2.5` 超阈值 | 蜂鸣器持续开启 |
| 无告警 | 显示 `Normal`，关闭蜂鸣器 |

补充：`beep_test_mode` 当前只是保留的报警屏蔽变量，不再由按键切换。

## 9. LCD 显示内容

摔倒检测相关显示项：

| 显示项 | 来源 |
|---|---|
| `System: Normal/FALL!!/PM2.5!!` | 主告警状态 |
| `State:` | `fall_detect_get_state_string()` |
| `Accel X/Y/Z` | `MPU_Get_Accelerometer()` |
| `MPU:OK/Error` | `MPU_Init()` 返回值 |

当前界面还同时显示 PMS7003、WiFi/Cloud 和补光状态，这些不属于本文维护范围。

## 10. 测试建议

1. 静止放置传感器，确认 `Accel X/Y/Z` 稳定，状态为 `Normal`。
2. 轻微移动，确认不频繁误报。
3. 在软垫上进行低风险模拟跌落测试，观察状态是否经过 `FreeFall/Impact/Stable/Detected`。
4. 若误报多，优先上调冲击阈值或延长稳定时间；若漏报多，优先下调自由落体/冲击判断门槛。
