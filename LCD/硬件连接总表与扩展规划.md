# 硬件连接总表与扩展规划

## 1. 维护范围

本文只维护硬件接线、引脚占用、预留资源和与当前源码不一致的标注。功能状态和软件流程以 `项目总结.md`、各模块说明文档为准。

## 2. 判定依据

本次整理按以下事实确认：

- `Core/Src/main.c`：当前主流程实际初始化和调用的外设。
- `Core/Src/gpio.c`、`Core/Inc/main.h`：当前 GPIO 宏定义和初始化。
- `Core/Src/usart.c`：当前串口初始化。
- `Drivers/HARDWARE/*`、`MyCode/*`：当前参与 Keil 工程的业务驱动。
- `MDK-ARM/LCD.uvprojx`：当前工程包含 `LCD/PMS7003/DS18B20/ESP8266/BEEP/KEY/FALL_DETECT/FILTER` 等驱动，未包含 `DHT11/` 和 `USART6` GPS 代码。

## 6. 已释放/历史资源

### 6.1 旧 FSMC LCD 资源

当前 `main.c` 不调用 `MX_FSMC_Init()`，Keil 工程也未加入 `Core/Src/fsmc.c`。旧并口 LCD 资源在当前 SPI 方案下视为释放：

- `PD0 / PD1 / PD4 / PD5 / PD8 / PD9 / PD10 / PD14 / PD15`
- `PE7 ~ PE15`
- `PF12`
- `PG12`
- `PB15`

说明：`Core/Src/fsmc.c` 和 `Core/Inc/fsmc.h` 仍在文件系统中，可视为历史备份文件，不代表当前硬件占用。

### 6.2 DHT11 历史残留

`DHT11/dht11.h` 定义 `PG9` 为 DHT11 数据脚，但该目录当前未加入 `MDK-ARM/LCD.uvprojx`，`main.c` 也未调用 `dht11_init()` 或 `dht11_read_data()`。因此：

- `DHT11/` 不作为当前功能统计。
- `PG9` 已作为 DS18B20 #1 使用，不能再按 DHT11 历史定义接线。
