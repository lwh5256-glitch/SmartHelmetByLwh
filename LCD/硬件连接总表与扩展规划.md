# 硬件连接总表与扩展规划

## 1. 判定依据

本表按以下三类信息交叉确认：

1. `MDK-ARM/LCD.uvprojx` 中实际参与编译的文件
2. `Core/Src/*.c`、`Drivers/HARDWARE/*`、`MyCode/*` 中真实生效的引脚定义
3. 已确认的最新硬件方案：
   - LCD 已切换为 `ATK-MD0240 SPI 接口`
   - 两个 DS18B20 改为两个独立 GPIO
   - 新增 `G5516` 光敏电阻模块
   - 新增 `LR7843` 场效应管模块，使用普通 GPIO 开关控制

## 2. 先说结论

### 2.1 当前项目最新外设分配

- LCD：`SPI1 + GPIO`
- MPU6050：软件 I2C
- PMS7003：`USART1`
- ESP8266：`USART3`
- GPS：预留 `USART6`
- DS18B20 #1：`PG9`
- DS18B20 #2：`PG10`
- G5516 光敏模块：`PC0`
- LR7843 场效应模块：`PC1`
- 蜂鸣器：`PF8`
- 按键：`PA0`、`PE2`、`PE3`、`PE4`
- LED1/LED2：`PF9`、`PF10`

### 2.2 关键变化

- LCD 不再使用 `FSMC + PB15`，改为 `SPI LCD`
- 两个 DS18B20 不再共用单总线，改为 `PG9` 和 `PG10` 独立接入
- 新增环境光检测与灯控链路：
  - `G5516 -> PC0`
  - `LR7843 -> PC1`

## 3. 当前工程最新引脚占用

### 3.1 核心应用外设

| 模块 | 引脚 | 说明 |
|---|---|---|
| KEY_UP | `PA0` | 高电平按下 |
| PMS7003 | `PA9 / PA10` | `USART1_TX / USART1_RX`，当前主用 RX 接收数据 |
| MPU6050 | `PB6 / PB7` | 软件 I2C，`SCL / SDA` |
| LCD_SPI | `PB3 / PB5` | `SPI1_SCK / SPI1_MOSI` |
| LCD_DC | `PB4` | SPI LCD 数据/命令控制 |
| ESP8266 | `PB10 / PB11` | `USART3_TX / USART3_RX` |
| G5516 | `PC0` | 光敏模块输入 |
| LR7843 | `PC1` | 灯控开关输出 |
| GPS | `PC6 / PC7` | `USART6_TX / USART6_RX` 预留 |
| KEY2 | `PE2` | 低电平按下 |
| KEY1 | `PE3` | 低电平按下 |
| KEY0 | `PE4` | 低电平按下 |
| 蜂鸣器 | `PF8` | 推挽输出 |
| LED1 | `PF9` | 状态灯 |
| LED2 | `PF10` | 心跳灯 |
| LCD_PWR | `PG6` | SPI LCD 电源/使能 |
| LCD_CS | `PG7` | SPI LCD 片选 |
| LCD_RST | `PG8` | SPI LCD 复位 |
| DS18B20 #1 | `PG9` | 单总线温度采集 |
| DS18B20 #2 | `PG10` | 单总线温度采集 |

### 3.2 当前不再占用的旧 LCD 引脚

原 FSMC LCD 释放资源：

- `PD0 / PD1 / PD4 / PD5 / PD8 / PD9 / PD10 / PD14 / PD15`
- `PE7 ~ PE15`
- `PF12`
- `PG12`
- `PB15`

这些引脚在“新硬件方案”下可重新规划，但当前文档暂未分配给新增模块。

### 3.3 保留说明

| 引脚 | 状态 | 说明 |
|---|---|---|
| `PA6` | 暂不分配 | 当前 `gpio.c` 中被配置为普通输出高电平，用途未在业务文档中明确，先保留，不分配给新模块 |
| `PG9` | 可直接用于 DS18B20 | 探索者板载单总线接口，通常已有 4.7K 上拉 |
| `PG10` | 可用于 DS18B20 | 需要按实际接线补 4.7K 上拉 |

## 4. 新增模块接线方案

## 4.1 SPI LCD

你已确认的最终接线如下：

| ATK-MD0240 模块 | 探索者 STM32F407 开发板 |
|---|---|
| `GND` | `GND` |
| `3V3` | `3.3V` |
| `PWR` | `PG6` |
| `CS` | `PG7` |
| `SCK` | `PB3` |
| `SDA` | `PB5` |
| `WR` | `PB4` |
| `RST` | `PG8` |

说明：

- 这组引脚不会和当前项目其他模块冲突
- 切换到 SPI 后，原 FSMC 引脚全部释放

## 4.2 两个 DS18B20

### 方案结论

两个 DS18B20 使用两个独立 GPIO：

- `DS18B20 #1 -> PG9`
- `DS18B20 #2 -> PG10`

### 接线表

| DS18B20 模块 | VDD | GND | DQ |
|---|---|---|---|
| `DS18B20 #1` | `3.3V` | `GND` | `PG9` |
| `DS18B20 #2` | `3.3V` | `GND` | `PG10` |

### 说明

- `PG9` 可以优先复用探索者板载单总线资源
- `PG10` 建议外接 `4.7K` 上拉到 `3.3V`
- 两个探头独立 GPIO 后，软件实现更直接，不需要做 ROM 搜索区分

## 4.3 GPS

### 推荐主方案

GPS 单独迁移到 `USART6`：

- `PC6 -> GPS_RXD`
- `PC7 -> GPS_TXD`

### 接线表

| GPS 模块引脚 | 连接到 |
|---|---|
| `VCC` | 按模块板卡要求供电 |
| `GND` | `GND` |
| `TXD` | `PC7`（MCU RX） |
| `RXD` | `PC6`（MCU TX） |

### 为什么继续保留这个方案

- 不和 `PMS7003(USART1)` 冲突
- 不和 `ESP8266(USART3)` 冲突
- 不占用新增灯控和温度采集资源
- `PC6 / PC7` 在当前工程中最干净

## 4.4 G5516 光敏电阻模块

### 接线方案

| G5516 模块引脚 | 连接到 |
|---|---|
| `VCC` | `3.3V` |
| `GND` | `GND` |
| `DO` | `PC0` |

### 说明

- 本方案按“一根引脚接入模块”来使用，默认使用模块的数字输出 `DO`
- 若你后续想做模拟采样，可以再改接 `AO` 到 ADC 引脚
- 当前版本先按“明暗阈值判断”设计，便于快速联动灯控

## 4.5 LR7843 场效应管模块

### 接线方案

| LR7843 模块引脚 | 连接到 |
|---|---|
| `VCC` | 按模块要求供电 |
| `GND` | `GND` |
| `IN` | `PC1` |

### 控制方式

- `PC1 = 1`：导通
- `PC1 = 0`：关闭

说明：

- 按你的要求，不使用 PWM
- 这里只做简单 GPIO 开关控制，用于控制灯的开与关

## 7. 建议的软件改动方向

### 7.1 DS18B20

- 新建 `Drivers/HARDWARE/DS18B20`
- 提供两个独立 GPIO 版本驱动
- 分别读取 `PG9` 和 `PG10` 上的温度
- 直接映射为“探头1 / 探头2”

### 7.2 GPS

- 新增 `USART6` 初始化与中断
- 先做 `NMEA` 接收解析
- 至少解析 `RMC / GGA`

### 7.3 光照联动灯控

- `PC0` 读取 `G5516` 数字输出
- `PC1` 控制 `LR7843`
- 根据外界光照做简单逻辑：
  - 环境暗 -> 开灯
  - 环境亮 -> 关灯

## 8. 最终建议

当前最稳妥、最完整的最新硬件方案是：

- LCD 使用 `SPI`
- 两个 DS18B20 分别走 `PG9 / PG10`
- GPS 保留 `USART6(PC6 / PC7)`
- 光照检测使用 `G5516 -> PC0`
- 灯控执行使用 `LR7843 -> PC1`
- `PA6` 暂时不分配，避免踩到历史配置

这版分配冲突最少，后续软件结构也最清晰。
