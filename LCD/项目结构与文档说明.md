# 项目结构与文档说明

## 1. 项目结构

| 目录/文件 | 当前作用 | 备注 |
|---|---|---|
| `Core/` | Cube/HAL 生成代码、主程序入口、外设初始化 | `main.c/gpio.c/usart.c/tim.c` 为当前重点 |
| `Drivers/HARDWARE/` | 项目业务驱动 | 当前包含 LCD、PMS7003、ESP8266、DS18B20、KEY、BEEP、FILTER、FALL_DETECT |
| `Drivers/SYSTEM/` | 正点原子系统支撑代码 | `delay/sys/usart` 等基础组件 |
| `Drivers/STM32F4xx_HAL_Driver/` | STM32 HAL 库 | 第三方库，不作为业务文档重点 |
| `Drivers/CMSIS/` | CMSIS 与芯片头文件 | 第三方库，不作为业务文档重点 |
| `MyCode/` | MPU6050、DMP、软件 I2C 支撑 | 当前仍在使用 |
| `DHT11/` | 历史残留 DHT11 驱动 | 当前未加入 Keil 工程、主流程未调用 |
| `MDK-ARM/` | Keil 工程与构建输出 | `LCD.uvprojx` 是工程配置；`MDK-ARM/LCD/` 为构建产物 |
| `资料/` | 外设手册、示例工程、参考资料 | 仅作资料来源，不等同于当前工程实现 |
| `LCD.ioc` | CubeMX 配置文件 | 当前源码已有手工改动，不能完全依赖 ioc 判断现状 |

## 2. 当前主要源码职责

| 路径 | 职责 |
|---|---|
| `Core/Src/main.c` | 系统主流程、界面绘制、传感器轮询、蜂鸣器告警、WiFi/云端流程、补光控制 |
| `Core/Src/gpio.c` | GPIO 初始化，包含光敏、补光、蜂鸣器、LED、`PB6/PB7` 软件 I2C 基础配置 |
| `Core/Src/usart.c` | `USART1(PMS7003)` 和 `USART3(ESP8266)` 初始化 |
| `Core/Src/tim.c` | `TIM5` 初始化，当前由系统初始化调用 |
| `Core/Src/stm32f4xx_it.c` | 中断入口，包含 USART1/PMS7003 和 USART3/ESP8266 接收分发 |
| `Drivers/HARDWARE/LCD/lcd.c` | 当前 ATK-MD0240 SPI LCD 驱动，提供 `lcd_*` 兼容接口 |
| `Drivers/HARDWARE/PMS7003/pms7003.c` | PMS7003 数据帧校验、解析和数据缓存 |
| `Drivers/HARDWARE/DS18B20/ds18b20.c` | 双通道 DS18B20 单总线温度检测，使用 `PG9/PG10` |
| `Drivers/HARDWARE/ESP8266/esp8266.c` | ESP8266 AT 指令、联网、SmartConfig、原子云通信 |
| `Drivers/HARDWARE/KEY/key.c` | 按键初始化和边沿扫描，当前业务只使用 `KEY_UP/KEY0` |
| `Drivers/HARDWARE/BEEP/beep.c` | 蜂鸣器开关、翻转和阻塞式报警 |
| `Drivers/HARDWARE/FALL_DETECT/fall_detect.c` | 摔倒检测状态机 |
| `Drivers/HARDWARE/FILTER/filter.c` | 滤波算法 |
| `MyCode/mpu6050.c` | MPU6050 读写、初始化和原始数据读取 |
| `MyCode/inv_mpu*.c` | DMP 支撑代码，当前作为 MPU 相关保留模块 |

## 4. 当前不一致标注规则

后续发现文档与源码不一致时，先按下面规则标注，不直接删除历史信息：

| 标注 | 含义 |
|---|---|
| 当前已落地 | 已在主流程或 Keil 工程中使用 |
| 规划/预留 | 文档已有方案，但当前源码未实现 |
| 驱动保留 | 驱动层存在，但主流程未使用 |
| 历史残留 | 文件存在，但当前工程未编译或未调用 |
| 待确认 | 源码有配置，但无法从业务逻辑确认用途 |

## 5. 文档维护规则

1. 修改硬件接线时，先更新 `硬件连接总表与扩展规划.md`。
2. 修改项目功能状态时，同步更新 `项目总结.md`。
3. 修改单个模块行为时，更新对应模块文档。
4. 历史迁移方案不当作当前实现说明，必须注明“已落地”或“历史记录”。
5. 每次文档整理都要写入 `项目改动日志.md`，并在被整理文档的“更新记录”中留下日期。
6. 不把 `资料/` 示例工程内容直接写成当前工程事实，除非能在源码或 Keil 工程中确认。
