# STM32F407 摔倒检测 + 空气质量监测系统

---

## 硬件平台

| 项目 | 说明 |
|------|------|
| MCU | `STM32F407ZGTx` |
| 开发板 | 正点原子 探索者 F407 |
| LCD | `ATK-MD0240` 2.4 寸 SPI LCD |

---

## 功能模块

| 功能 | 状态 | 说明 |
|------|------|------|
| LCD显示 | 已规划为 SPI 方案 | 使用 `SPI1 + GPIO` |
| 摔倒检测 | 已实现 | 状态机：`Normal -> FreeFall -> Impact -> Stable -> FALL` |
| PM2.5监测 | 已实现 | `USART1` 接收，超标报警 |
| 粉尘浓度换算 | 已实现 | `dust = PM2.5 / 1000.0` |
| 蜂鸣器报警 | 已实现 | 摔倒/PM2.5 超标触发 |
| 按键控制 | 已实现 | 4 个按键 |
| WiFi连接 | 已实现 | ESP8266 + 原子云 |
| GPS | 已规划 | `USART6` |
| DS18B20 x2 | 已规划 | 两路独立 GPIO |
| 环境光联动灯控 | 已规划 | `G5516 + LR7843` |

## 注意事项

1. LCD 已从旧 `FSMC` 方案切换到 `SPI` 接线文档，旧 `PB15 / FSMC` 说法全部作废。
2. `DS18B20 #2` 使用 `PG10` 时，建议外接 `4.7K` 上拉到 `3.3V`。
3. `G5516` 当前按数字输出 `DO` 使用，默认只做明暗阈值判断。
4. `LR7843` 当前按普通 GPIO 开关控制，不使用 PWM。
5. `PA6` 在现有代码中已被拉高，但用途未在业务逻辑中明确，暂不建议复用。
